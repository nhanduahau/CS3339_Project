## Introduction

This repository contains a collection of CPU benchmarks written in C++. Each benchmark targets a different workload pattern so you can compare single-core behavior, multi-core scaling, memory pressure, and branch-heavy execution.

## Benchmark Objectives

The suite is designed to measure:

- Single-thread performance
- Multi-thread scaling
- Memory and cache behavior
- Branch-heavy control flow
- Floating-point and vector-friendly workloads

## Experimental Notes

- Every benchmark accepts the same runtime interface: mode and runs.
- In single-core mode, the active thread is pinned to core 0.
- In multi-core mode, all hardware threads are used.
- Use the same compiler and optimization flags across systems for fair comparison.

## Included Benchmarks

### 1. Basic Math (basicmath.cpp)

- Workload: Repeated sin/cos/sqrt/log/tanh operations over a large iteration space.
- Focus: Floating-point throughput and sustained FPU pressure.

### 2. Bit Count (bitcount.cpp)

- Workload: Counts set bits in transformed 64-bit values using Brian Kernighan style popcount loops.
- Focus: Integer bitwise throughput, loop behavior, and branch/control flow.

### 3. CRC32 (CRC32.cpp)

- Workload: Computes CRC-32 checksums over generated byte buffers using the bit-by-bit polynomial update method.
- Focus: Bitwise arithmetic and large sequential memory processing.

### 4. Dijkstra (dijkstra.cpp)

- Workload: Builds a large random weighted graph and runs repeated shortest-path queries.
- Focus: Memory-access patterns, heap operations, and irregular control flow.

### 5. Fast Fourier Transform (FFT.cpp)

- Workload: Iterative radix-2 Cooley-Tukey FFT on large complex arrays.
- Focus: Floating-point math, cache behavior, and vectorization-friendly computation.

### 6. JPEG-like DCT (jpeg.cpp)

- Workload: Repeated 8x8 DCT blocks with a lightweight quantization-like step.
- Focus: Dense numeric kernels and multi-thread scaling for image-style transforms.

### 7. Sorting Stress (qsort.cpp)

- Workload: Repeated std::sort passes over large arrays (random, reversed, then remapped values).
- Focus: Data-dependent branching, memory traffic, and sort throughput.

### 8. String Search (stringsearch.cpp)

- Workload: Generates large random text buffers and repeatedly searches fixed substring patterns.
- Focus: String scanning cost, cache locality, and branch behavior in search loops.

## Requirements

- C++17 compiler (g++, clang++, or MSVC)
- Python 3 (optional, for compile_all.py)

## Build

### Option A: Automated Build (recommended)

The script compiles every .cpp file in the folder and creates .exe output names.

```bash
python compile_all.py
```

Force a clean rebuild:

```bash
python compile_all.py --recompile
```

### Option B: Manual Build (Linux, macOS, MinGW)

Use consistent optimization flags. Since all benchmarks use std::thread, include -pthread for g++/clang++ on POSIX systems.

```bash
g++ -std=c++17 -O3 -march=native -pthread -o basicmath.exe basicmath.cpp
g++ -std=c++17 -O3 -march=native -pthread -o bitcount.exe bitcount.cpp
g++ -std=c++17 -O3 -march=native -pthread -o CRC32.exe CRC32.cpp
g++ -std=c++17 -O3 -march=native -pthread -o dijkstra.exe dijkstra.cpp
g++ -std=c++17 -O3 -march=native -pthread -o FFT.exe FFT.cpp
g++ -std=c++17 -O3 -march=native -pthread -o jpeg.exe jpeg.cpp
g++ -std=c++17 -O3 -march=native -pthread -o qsort.exe qsort.cpp
g++ -std=c++17 -O3 -march=native -pthread -o stringsearch.exe stringsearch.cpp
```

## Run

Each executable takes two arguments:

- mode: 1 for single-core pinned mode, 2 for multi-core mode
- runs: integer from 1 to 10

Examples:

```bash
./basicmath.exe 1 3
./basicmath.exe 2 5
```

On Windows PowerShell:

```powershell
.\basicmath.exe 1 3
.\basicmath.exe 2 5
```

## Output

Each benchmark prints:

- Selected mode and thread count
- Per-run elapsed time
- Mean elapsed time
- Final checksum/result value

The checksum is a correctness guard to reduce the chance of dead-code elimination and to verify deterministic workload behavior.
